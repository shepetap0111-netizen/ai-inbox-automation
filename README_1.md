# AI-обработка входящих заявок (Email + Telegram)

Автоматизация на **n8n**, которая принимает обращения из Gmail и Telegram, классифицирует их с помощью LLM, пишет клиенту готовый ответ и логирует всё в Google Sheets как мини-CRM.

> Демо-проект, показывающий полный цикл AI-автоматизации: приём сообщений → классификация → ответ клиенту → запись в CRM, с отказоустойчивостью и дедупликацией.

---

## Что делает

- **Собирает** входящие обращения из двух каналов — Gmail и Telegram — в единый поток.
- **Классифицирует** каждое сообщение через LLM (Groq, Llama 3.3): определяет категорию (продажа / поддержка / спам / другое), приоритет и настроение клиента.
- **Отвечает** клиенту — модель генерирует осмысленный ответ по сути обращения, а не шаблон.
- **Приоритизирует** — срочные заявки (high) мгновенно уведомляют администратора в Telegram.
- **Логирует** результат в Google Sheets: категория, приоритет, суть, ответ, контакт клиента.
- **Отсекает дубли** — повторные обращения не создают лишних строк.

---

## Как это работает

```
Gmail ──┐
        ├─→ Нормализация ─→ Merge ─→ Groq (LLM) ─→ Парсинг ─→ Sheets ─→ Приоритет ─→ Ответ
Telegram┘                       │                                          │
                                └─(при отказе)─→ Fallback (правила)         └─→ Уведомление админа
```

1. **Триггеры** Gmail и Telegram ловят новые сообщения.
2. **Нормализация** приводит оба источника к единому формату (source, contact, text, chat_id...).
3. **Merge** объединяет потоки.
4. **Groq (LLM)** классифицирует и пишет ответ, возвращая структурированный JSON.
5. **Fallback** — если LLM недоступен, срабатывает классификация по правилам (ключевым словам). Поток не падает.
6. **Google Sheets** — запись с дедупликацией по уникальному ключу.
7. **Роутинг** — ответ уходит клиенту в его канал (Telegram/Email), срочные заявки уведомляют админа.

## Стек

- **n8n** — оркестрация воркфлоу
- **Groq API (Llama 3.3-70b)** — LLM-классификация и генерация ответов
- **Google Sheets** — хранение заявок (CRM)
- **Gmail API / Telegram Bot API** — источники и каналы ответа

---

## Ключевые технические решения

- **Структурированный вывод LLM** — модель возвращает строгий JSON (category, priority, sentiment, summary, reply), который напрямую маппится в поля воркфлоу.
- **Graceful degradation** — при сбое LLM поток переключается на rule-based классификацию, а не останавливается. В таблице фиксируется, каким методом обработано сообщение (llm / rules).
- **Дедупликация** — запись в Sheets в режиме appendOrUpdate по уникальному ключу из источника, контакта и текста. Повторы не плодят строки.
- **Единый формат из разных источников** — Email и Telegram нормализуются к одной схеме, дальше обрабатываются единообразно.

## Структура данных (Google Sheets)

| Поле | Описание |
|------|----------|
| timestamp | Время обращения |
| source | Канал (telegram / email) |
| from | Имя клиента |
| text | Текст обращения |
| category | support / sales / spam / other |
| priority | high / medium / low |
| sentiment | positive / neutral / negative |
| summary | Краткая суть |
| method | Как обработано (llm / rules) |
| contact_id | chat_id или email для связи |
| username | @ник в Telegram |
| reply | Ответ, сгенерированный моделью |

---

## Запуск

1. Импортировать `workflow.json` в n8n (Import from File).
2. Подключить credentials:
   - **Groq** — Header Auth (`Authorization: Bearer <api_key>`), ключ на console.groq.com
   - **Google Sheets** — OAuth2
   - **Gmail** — OAuth2
   - **Telegram** — Bot Token
3. Указать ID своей Google-таблицы в узле записи и создать заголовки колонок (см. таблицу выше).
4. Активировать воркфлоу.

## Возможные доработки
